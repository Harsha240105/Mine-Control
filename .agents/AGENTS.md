# Project Rules

1. **Update README**: Every time a given task is completed, update the `README.md` file. Provide a total summary about the project's works end-to-end in the `README.md` file every time code is pushed to GitHub.
2. **Release Versioning**: Only release a new version if the user explicitly specifies the release version. If no release version is mentioned, DO NOT release a version or push tags.
